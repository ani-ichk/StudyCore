# Работа с uv
Как установить uv на ваш компьютер: https://docs.astral.sh/uv/getting-started/installation/

Дополнительная инструкция: https://habr.com/ru/articles/875840/

Важно! Многие IDE (PyCharm, VS Code) автоматически создают виртуальное окружение при открытии проекта. Проверьте, не создалось ли окружение автоматически. Если да — удалите его, чтобы избежать конфликтов. После удаления выполните эту команду для создания виртуального окружения через uv с Python 3.12: uv venv --python=3.12

Выберите созданное окружение как интерпретатор Python:
- PyCharm: Settings → Project → Python Interpreter → Add → Existing environment → выбрать .venv/bin/python (или  venv\Scripts\python.exe на Windows)
- VS Code: Ctrl+Shift+P → Python: Select Interpreter → выбрать путь к .venv

Далее нужно установить все нужные зависимости, прописав команду: uv sync

Управление зависимостями
- Для того, чтобы добавить библиотеку в проект: uv add <библиотека>
- Для того, чтобы удалить добавленную библиотеку из проекта: uv remove <библиотека>

После добавления/удаления библиотек:
- Автоматически обновляется pyproject.toml и uv.lock
- Обязательно коммитьте эти файлы в репозиторий

Важные замечания
- .venv/ — виртуальное окружение, не коммитить (должно быть в .gitignore, git автоматически игнорирует .venv)
- pyproject.toml и uv.lock — обязательно коммитить
- При смене веток всегда выполняйте uv sync для синхронизации зависимостей
- Если возникают конфликты с окружением IDE — удалите его и создайте через uv заново
